# SUBFASE 4.1.1 — RUNTIME LOOP FOUNDATION

## OBJETIVO REAL

Construir el loop base controlado del runtime Hermes.

Esta subfase crea:
la fundación operacional del runner.

IMPORTANTE:

Todavía NO se ejecutan tasks.

Todavía NO existe integración IA.

Todavía NO existe orchestration avanzada.

El objetivo es:
tener un runtime loop estable,
observable
y controlado.

━━━━━━━━━━━━━━━━━━━
REGLA CRÍTICA
━━━━━━━━━━━━━━━━━━━

El runtime loop NO puede convertirse en un loop agresivo o descontrolado.

El loop debe ser:
- predecible
- observable
- estable
- reversible

Hermes controla:
- frecuencia
- estado alive
- ciclo operacional
- pausas runtime

━━━━━━━━━━━━━━━━━━━
OBJETIVO TÉCNICO
━━━━━━━━━━━━━━━━━━━

Construir el loop base reutilizable que será usado después por:

* task pickup
* execution pipeline
* orchestration
* retries
* monitoring
* recovery

SIN mezclar responsabilidades futuras.

━━━━━━━━━━━━━━━━━━━
IMPLEMENTAR
━━━━━━━━━━━━━━━━━━━

# 1. Runtime Loop Base

Crear loop operacional base.

Responsabilidades:

* iniciar ciclo runtime
* mantener alive state
* ejecutar heartbeat
* controlar timing
* permitir stop controlado

SIN ejecutar tasks todavía.

━━━━━━━━━━━━━━━━━━━

# 2. Alive State Control

Implementar estado runtime:

* alive = true
* alive = false

Debe reflejar:
estado REAL del loop.

PROHIBIDO:
alive falso positivo.

━━━━━━━━━━━━━━━━━━━

# 3. Heartbeat Runtime

Implementar heartbeat simple.

Objetivo:

confirmar que el loop:
- sigue vivo
- sigue operativo
- sigue respondiendo

SIN lógica compleja todavía.

━━━━━━━━━━━━━━━━━━━

# 4. Loop Timing Control

Implementar timing controlado:

* sleep runtime
* intervalos controlados
* frecuencia estable

PROHIBIDO:

* busy loops
* polling agresivo
* consumo innecesario CPU

━━━━━━━━━━━━━━━━━━━

# 5. Controlled Stop

Implementar stop seguro del loop.

Debe permitir:

* detener runtime
* preservar estado
* cerrar loop limpiamente

SIN corrupción operacional.

━━━━━━━━━━━━━━━━━━━

# 6. Runtime State Visibility

Exponer:

* alive
* started_at
* last_heartbeat
* loop_iteration

Datos REALES solamente.

━━━━━━━━━━━━━━━━━━━

# 7. Minimal Runtime Logging

Loggear solamente:

* runtime started
* runtime stopped
* heartbeat
* runtime errors críticos

SIN spam logs.

━━━━━━━━━━━━━━━━━━━

# 8. Error Containment Base

Si el loop encuentra error:

* preservar runtime state
* evitar crash total
* loggear error
* mantener observabilidad

PROHIBIDO:
crash silencioso.
