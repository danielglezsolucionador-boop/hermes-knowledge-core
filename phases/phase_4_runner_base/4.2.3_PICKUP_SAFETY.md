# SUBFASE 4.2.3 — PICKUP SAFETY

## OBJETIVO REAL

Construir la capa de protección operacional del pickup runtime Hermes.

Esta subfase protege el sistema contra:

- race conditions
- doble pickup
- corrupción estado
- loops agresivos
- ownership inválido
- inconsistencias runtime

Esta subfase NO ejecuta tasks todavía.

# 1. Pickup Safety Layer

Crear sistema de protección operacional del pickup.

Responsabilidades:

- validar ownership
- detectar conflictos
- proteger runtime
- evitar duplicados
- contener errores pickup

━━━━━━━━━━━━━━━━━━━

# 2. Duplicate Pickup Prevention

Implementar protección contra:

- doble claiming
- doble pickup
- múltiples runners sobre misma task
- ownership simultáneo

Debe existir:

- verificación previa
- validación runtime
- control ownership

━━━━━━━━━━━━━━━━━━━

# 3. Race Condition Protection

Preparar protección contra:

- runners simultáneos
- polling overlap
- state conflicts
- stale ownership

PROHIBIDO:

- updates inseguros
- ownership ambiguo
- sincronización falsa

━━━━━━━━━━━━━━━━━━━

# 4. Stale Claim Detection

Detectar claims inválidos:

- ownership expirado
- runtime muerto
- heartbeat perdido
- claims huérfanos

Debe existir:

- validación runtime
- control temporal
- observabilidad

━━━━━━━━━━━━━━━━━━━

# 5. Runtime Consistency Validation

Validar:

- ownership consistente
- runtime válido
- task válida
- claim válido
- state coherente

NO permitir:

- estados corruptos
- ownership inválido
- transiciones inconsistentes

━━━━━━━━━━━━━━━━━━━

# 6. Safe Pickup Retry Control

Preparar retry controlado SOLO para pickup.

IMPORTANTE:

NO retries execution todavía.

SOLO:

- retry pickup mínimo
- retry limitado
- retry observable

Evitar:

- retry loops
- spam retries
- runtime saturation

━━━━━━━━━━━━━━━━━━━

# 7. Pickup Error Containment

Si ocurre error:

- contener fallo
- preservar runtime
- evitar crash total
- loggear conflicto
- continuar estable

PROHIBIDO:

- crash silencioso
- corrupción runtime
- reinicio agresivo

━━━━━━━━━━━━━━━━━━━

Loggear:

- ownership conflicts
- duplicate prevention
- stale claims

- pickup errors
