# OPERATIONAL_RULES

1. Stability before expansion.
2. Runtime integrity is priority.
3. No uncontrolled architecture growth.
4. No undocumented systems.
5. Every application must have phases.
6. Every phase must have validation.
7. Operational simplicity over complexity.
8. Recovery capability is mandatory.
9. No application connects directly to runtime core without validation.
10. Knowledge Core is source of operational truth.
