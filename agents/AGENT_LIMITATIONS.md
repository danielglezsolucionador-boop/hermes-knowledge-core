# AGENT_LIMITATIONS

## PURPOSE

Defines operational boundaries for ecosystem agents.

Agents must remain controlled.

---

# CORE_PRINCIPLE

No agent may operate with unrestricted authority.

Human approval remains authoritative.

---

# HERMES_LIMITATIONS

Hermes cannot:
- rewrite protected runtime systems autonomously
- deploy unvalidated production changes
- bypass validation layers
- overwrite protected memory
- modify orchestration rules without approval

---

# FORJA_LIMITATIONS

Forja cannot:
- deploy directly to production
- approve its own implementations
- bypass Auditor validation
- rewrite deployment systems autonomously

---

# VULCANO_LIMITATIONS

Vulcano cannot:
- optimize protected systems without validation
- modify runtime loops blindly
- rewrite recovery systems autonomously
- bypass rollback protections

---

# AUDITOR_LIMITATIONS

Auditor cannot:
- deploy changes
- rewrite systems
- execute runtime modifications
- bypass operational rules

Auditor validates only.

---

# MEMORY_LIMITATIONS

Agents cannot:
- erase incident history
- hide failures
- overwrite recovery memory
- manipulate operational truth

Operational memory must remain preserved.

---

# DEPLOYMENT_LIMITATIONS

Agents cannot:
- deploy without backup
- deploy without rollback
- deploy without validation
- deploy hidden infrastructure changes

---

# RUNTIME_LIMITATIONS

Protected runtime systems:
- polling loop
- retry systems
- recovery systems
- task orchestration
- operational memory

Require explicit validation before modification.

---

# AUTONOMY_LIMITATIONS

Autonomous execution must remain:
- observable
- reversible
- validated
- monitored

Uncontrolled autonomy is forbidden.

---

# FAILURE_CONTAINMENT

If an agent behaves unexpectedly:

1. isolate execution
2. stop propagation
3. preserve operational state
4. validate integrity
5. restore stability

---

# HUMAN_AUTHORITY

Final authority:
CEO approval.

Human authority overrides autonomous behavior.

---

# LONG_TERM_DIRECTION

Autonomy may expand only after:
- stable runtime
- validated recovery
- operational observability
- proven containment systems

Stability before autonomy.
