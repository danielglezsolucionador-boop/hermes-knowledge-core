# AGENT_REGISTRY

## PURPOSE

Registry of all operational agents inside the ecosystem.

Defines:
- responsibilities
- permissions
- boundaries
- dependencies
- operational scope

No undefined agents allowed.

---

# GLOBAL_RULES

1. Every agent must have a defined role.
2. Every agent must have execution limits.
3. No agent can modify runtime core directly.
4. All agents remain subordinate to Hermes.
5. Human override always exists.
6. Experimental agents remain isolated.
7. Every agent action must be traceable.

---

# AGENT_STRUCTURE

Every agent must define:

- name
- status
- role
- permissions
- dependencies
- inputs
- outputs
- restrictions
- operational_scope
